mineucore-Buddy system

Buddy system将物理内存组织为了一颗满二叉树，每个叶子节点维护一个物理页，叶子节点的个数为2的整数次幂

1、分配阶段

在2^m + 2^(m-9) 与 n 中找到平衡，最大化利用内存。

假设满二叉树共维护2^m个物理页，那么满二叉树的节点个数为2^m x 2 -1，取为2^(m+1)，每个节点需要4字节来记录其可用空间，那么维护满二叉树需要的总的存储空间为2^(m+3)字节，因为内存需要页对齐，一页的大小为2^12字节，所以
维护满二叉树需要的物理页数为max{1,2^(m-9)}。

当内存n小于等于512页时，其节点信息至少占用一页，想要最大化利用内存，就是在2^m +1 与n之间找到平衡点，那么最好的平衡点就是2^m = ⌈n-1⌉，这里的向上取整是指向上取2的整数次幂。

当内存n大于512页时，想要最大化利用内存，就是在2^m + 2^(m-9) 与n之间找到平衡点

  • 当2^m + 2^(m-9) >= n，这是最好的情况，内存可以被完全利用
  • 当2^m + 2^(m-9) < n <= 2^m + 2^(m-8)，这是内存有剩余，但是可以增加存储节点的页数，并不能真正增加实际物理页数
  • 当2^m + 2^(m-8) < n，此时同时可以增加存储节点信息的页数和实际物理页数

为了尽可能的使所有物理页都能得到利用，对于n>512的情况，在分配阶段，直接建立最大规模的二叉树，即2^m = ⌈n-1⌉，这里的向上取整也是向上取最大2的整数次幂，此时若2^m + 2^(m-9) >= n，则完成分配，若2^m + 2^(m-8) < n，则将维护页和分配页同时扩大2倍，使2^(m+1) + 2^(m-8) >= n，完成分配。对于二叉树中无法维护到物理页（即真实并不存在的物理页），可以视为已经被分配，且永远不会被释放

2、初始化阶段

初始化分配页，将不存在的物理页设为已分配，设v为虚拟分配页大小，r为真实分配页大小，具体递归操作如下

若v=r，则将空间设为空闲页
若r=0，则将空间初始话为已分配
若r<=v/2，则将右半空间初始化为已分配，v = v/2，继续在左半空间判断r与v的大小，进行操作
若r>=v/2，则更新v=v/2，r=r-v/2，继续在右半空间判断r与v的大小
